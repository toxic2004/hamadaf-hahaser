# הפעלת תוכנית השדרוג

הקוד הסטטי אינו זקוק לסודות. ערך ה Supabase שב `supabase-client.js` הוא מפתח publishable שהיה קיים ביישום ואינו מפתח service role. אין להכניס למאגר מפתחות פרטיים.

## מיפוי הנתונים

טבלת `books` הקיימת היא מקור הספרים ומכילה את מזהה המשתמש, שם הספר, מחבר, כריכה, הערות, סטטוס ותאריכי יצירה ועדכון. המיגרציות מוסיפות לה שדות בלבד:

- `isbn`
- `priority`, `is_favorite`, `is_required`
- `acquired_at`, `purchase_price`, `new_price`

מיגרציית המחירים מוסיפה טבלאות נפרדות:

- `price_offers` להצעות פעילות והזנה ידנית
- `price_history` לתיעוד הצעה פעם אחת בכל יום
- `daily_book_prices` להצעה הזולה ביותר לספר בכל יום
- `notifications` להתראות ולמניעת התראה כפולה
- `notification_settings` להגדרות נפרדות לכל משתמש
- `price_scan_runs` למניעת הפעלה כפולה של בדיקת בוקר או ערב

כל הטבלאות החדשות מפעילות RLS ומגבילות גישה ל `auth.uid()`. הפונקציה `snapshot_daily_prices` זמינה רק ל `service_role`. אין במיגרציות מחיקת טבלאות, מחיקת עמודות או שינוי בלתי הפיך של נתונים קיימים.

## סדר הפעלת המיגרציות

יש להריץ ב Supabase SQL Editor לפי הסדר:

1. `supabase-isbn-migration.sql`
2. `supabase/003_priority_favorites.sql`
3. `supabase/004_book_analytics.sql`
4. `supabase/005_prices_history_notifications.sql`

הפקודות משתמשות ב `if not exists` או בבדיקת קיום לפני הוספה, ולכן הן מתאימות להפעלה חוזרת ובטוחה. לפני הפעלה בסביבת ייצור מומלץ לבצע גיבוי רגיל של מסד הנתונים ולבדוק את המיגרציות בסביבת staging.

## פריסת ההתראות המתוזמנות

ההתראות המיידיות בתוך המערכת נוצרות בעת שמירת הצעה משתלמת או ירידת מחיר. כדי להפעיל גם בדיקות לפי השעות שנשמרו לכל משתמש באזור הזמן `Asia/Jerusalem`, דוח בוקר ומייל אופציונלי:

1. צור ב Supabase אסימון גישה אישי. שמור אותו ישירות ב GitHub ואל תכניס אותו לקוד או לצ'אט.
2. הוסף ב GitHub תחת `Settings`, אחר כך `Secrets and variables`, אחר כך `Actions`, את שני הסודות הבאים:

- `SUPABASE_ACCESS_TOKEN`, אסימון הגישה האישי מ Supabase
- `SUPABASE_PROJECT_REF`, מזהה הפרויקט שמופיע בכתובת הפרויקט ב Supabase

3. פתח ב GitHub את `Actions`, בחר `Deploy alerts function` ולחץ `Run workflow`.
4. לאחר ההפעלה הידנית הראשונה, כל שינוי עתידי בפונקציית `alerts` שייכנס לענף `main` ייפרס אוטומטית. התהליך מונע שתי פריסות מקבילות ומפסיק עם הודעה ברורה אם אחד הסודות חסר.
5. הגדר ב Supabase סוד אקראי בשם `ALERTS_SCHEDULE_SECRET`.

לפריסה מקומית חלופית בלבד אפשר להריץ:

```bash
supabase functions deploy alerts --project-ref PROJECT_REF --no-verify-jwt
```

6. הוסף ל GitHub Actions את הסודות הדרושים להפעלה השעתית:

- `SUPABASE_ALERTS_URL`, כתובת ה endpoint של פונקציית `alerts`
- `SUPABASE_ALERTS_SCHEDULE_SECRET`, אותו ערך שהוגדר בפונקציה

ה workflow מפעיל את ה endpoint פעם בשעה. הפונקציה ממשיכה לכל משתמש רק כאשר השעה בירושלים תואמת לשעת הבוקר או הערב שנשמרה עבורו, ושומרת run ייחודי למשתמש, תאריך וסוג בדיקה.

## בדיקת הפעלה ידנית

לאחר שהפונקציה נפרסה ושני סודות ההפעלה השעתית הוגדרו ב GitHub, אפשר לבצע בדיקה חיה מבוקרת:

1. פתח ב GitHub את `Actions` ובחר `Test deployed alerts`.
2. לחץ `Run workflow`, הקלד `RUN` בשדה האישור והפעל.
3. הבדיקה מוודאת תחילה שסוד שגוי נדחה, ואז מפעילה פעם אחת את בדיקת ההתראות עם הסוד התקין.
4. התהליך מציג רק מספר משתמשים ותוצאות. הוא אינו מדפיס סודות, כתובות מייל או פרטי משתמשים.

הבדיקה אינה מתוזמנת ואינה פועלת מעצמה. הפעלה עם הסוד התקין מבצעת בדיקת התראות אמיתית ועלולה ליצור התראות בהתאם לשעה ולהגדרות המשתמש. אם מיילים כבר הופעלו, היא עשויה גם לשלוח את ההתראות שנוצרו.

## מייל אופציונלי

שליחת מייל משתמשת ב Resend רק כאשר הוגדרו בפונקציית Supabase:

- `RESEND_API_KEY`
- `ALERTS_EMAIL_FROM`

בנוסף המשתמש צריך להפעיל מייל במסך ההתראות ולהזין כתובת. ללא כל ההגדרות האלה המערכת ממשיכה להציג התראות בתוך היישום ואינה מציגה מצג שווא שמייל נשלח.

## מקורות מחירים

מסך המחירים מספק קישורי חיפוש ליד2, סימניה, Marketplace, פייסבוק ציבורי, עברית, סטימצקי, צומת ספרים, חנויות עצמאיות וחיפוש כללי. איסוף אוטומטי אינו מופעל משום שחלק מהמקורות דורשים התחברות או מגבילים גישה. אין לעקוף הגנות, CAPTCHA או תנאי שימוש. משלימים בטופס את מחיר הספר, המשלוח, המיקום, מצב הספר, המוכר והקישור.

דירוג העסקה מתועד בממשק ומוגבל ל 100 נקודות: עד 60 לפי אחוז החיסכון מול מחיר חדש, 10 למחיר כולל ידוע, עד 15 לאיכות ההתאמה, 10 למהדורה בעברית ו 5 להצעה שנבדקה ביומיים האחרונים. מהדורה באנגלית או הצעה שסומנה כלא התאמה מקבלות ציון 0.
